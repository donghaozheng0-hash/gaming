## ★ 四条底层逻辑（用户钦定，凌驾于一切实现细节之上）

以下四条是《符塔》的产品宪法。任何设计、任务、实现、验收发生冲突时，**以这四条为最高优先级**。每条都给出了可被 Codex 执行、可被验收量化的落地约束。

### 逻辑一：摸鱼场景 · 单局轻负担 · 核心操作不复杂

- **场景假设**：玩家在通勤、排队、工作摸鱼的碎片时间单手竖屏游玩。任何设计都要扛得住“随时被打断、随时能回来”。
- **硬约束**：
  - 单局主线普通关 75–105 秒、Boss 关 110–140 秒（见总设计 6 章），不得超时。
  - 核心循环只有三件套：**摆位（拖拽）/ 对位（选带哪 5 符）/ 画功（可选手绘）**。除此之外不得新增“必须实时操作”的硬核负担。
  - 自动战斗为默认态：手残/摸鱼玩家**全程不画符也能稳定推图**，零惩罚（画符是上限不是门槛）。
  - 单局必做的主动决策点 ≤ 4 个（布阵 1 次 + 灵机点 3 次）。不得堆砌入口和弹窗。
  - 新手前 3 分钟遵循“先赢一把再教系统”（FTUE，总设计 1·C），绝不一上来糊脸 10 个入口。
- **反面红线**：禁止引入连续 QTE、长按蓄力链、多指同时操作、强制不可暂停的高频微操。

### 逻辑二：一定要爽 · 三类玩家都有爽点

- **肝帝爽**：等级不封顶 + 无尽符塔 + 五行专精圆满，永远有可追的数值与排名（总设计 3.6 / 3.7）。
- **氪佬爽**：8 条养成线堆战力，越级碾压低战力内容的“数值爽”成立；但**红线：不卖一锤定音的数值碾压**（总设计 8.3）。付费买的是效率与外观，不是把免费玩家锁死。
- **免费/技术玩家爽（翻盘空间）**：靠**相生摆位 + 克制对位 + 画功**，同一张符输出可差 3 倍（总设计 5.1）；技巧最多补 **25–30%** 战力差（总设计 3.6 / 4.6），保证越级有活路、不被纯数值碾退游。
- **设计哲学一句话**：**战力打底（下限），技巧翻盘（上限）。** 实现与验收时，下限（数值咬合）和上限（技巧收益）都必须可被量化复现。

### 逻辑三：数值与代码隔离 · 配置不得硬编码

- **铁律**：数值配置、关卡配置、符/怪/经济/融合/视觉色板配置，**一律不得硬编码进业务逻辑**。业务代码只读配置、跑公式，不内嵌具体数字。
- 所有可调参数（战斗参数、20 关、怪物模板、符模板、升级消耗、融合配方、推荐战力曲线、减伤常数 K=600、爬塔底数 1.15 等）必须落在独立的配置层。
- 详细架构、目录、Schema、校验门禁见 `docs/DATA_ARCHITECTURE.md`。这是**强制门禁项**，违反即判 FAIL。

### 逻辑四：雾山五行水墨国风 · 特效走水墨/泼墨/符箓/五行色彩

- 整体 UI 与世界观美术走《雾山五行》方向：水墨、留白、线条骨法、国风。
- 技能特效统一为水墨/泼墨/符箓笔触 + 五行色彩语言（金白、木青、水墨蓝、火朱、土赭）。
- 视觉是品牌护城河，不是后期换皮。详细色板、特效规范、验收标准见 `docs/VISUAL_DIRECTION.md`。

## 工作原则

1. Claude 先把用户想法转成可执行任务、验收标准和优先级。
2. Codex 按任务实现，不自行扩大范围。
3. 每批实现完成后必须运行 `npm run quality:gate`。
4. Claude 的验收结论必须引用门禁输出、截图/录像证据、文件变更和未解决风险。
5. “Claude 说通过”不等于通过；通过必须满足 `docs/ACCEPTANCE_GATE.md` 的核准值。
