# 符塔 · 视觉方向（雾山五行 · 水墨国风）

> 对应底层逻辑四：**整体 UI 走雾山五行水墨国风，技能特效走水墨/泼墨/符箓/五行色彩。**
> 维护：Claude（方向与验收标准）。实现：Codex（Babylon 渲染、shader、特效、UI）。
> 视觉是品牌护城河，不是后期换皮——从第一个可玩原型起就要带水墨气质。

## 1. 总基调

参考《雾山五行》：**水墨、留白、骨法用笔、国风写意**。不是写实 3D，是“会动的水墨画”。

- 大量留白与负空间，画面不要塞满。
- 线条优先：边缘有毛笔飞白/枯笔质感，而非硬边几何。
- 雾、烟、墨晕营造层次（远景淡、近景浓）。
- 动效讲究“起笔—行笔—收笔”的节奏感，泼墨炸开后有墨滴飞溅与晕染回收。

## 2. 五行色板（写入 `src/config/visual.json`，业务/特效只引用色键）

| 五行 | 色名 | 主色（建议起点，可真机调） | 用于 |

| 合成·雷 | 紫电 | `#7A5BAE` | 连锁跳弹 |

| 合成·毒/瘴 | 黯绿 | `#6B8E3D` | DoT 瘴气 |

- 强调/警示（阵眼告急）：朱砂红描边 + 墨晕扩散，不用刺眼的霓虹。

## 3. 技能特效语言（统一规范）

| 元素 | 特效母题 | 表现要点 |
|---|---|---|
| 通用施法 | 符箓 | 出手时浮现符箓纹样（朱砂笔迹），亮起后笔画化为攻击 |
| 火 | 泼墨焚烧 | 朱赤泼墨炸开 + 飞白火舌，边缘有焦墨 |
| 水 | 墨晕扩散 | 蓝墨在“纸面”上晕染推开，冰棱为锐利留白线条 |
| 木 | 飞白藤生 | 枯笔藤蔓沿地面生长，减速时画面边缘起青雾 |
| 金 | 一笔斩 | 单线锋银快斩，破盾时墨片崩裂飞溅 |
| 土 | 浓墨砸落 | 落岩为延迟蓄力的浓墨团坠地泼溅，震屏 |
| 雷 | 焦墨电纹 | 紫电以折线笔画连点，跳弹有顿挫感 |
| 画符成功 | 笔迹追光 | 玩家描的笔迹被点亮、收笔时墨花绽放（≥80% 触发隐藏泼墨大特效） |

- 命中/暴击：墨点飞溅 + 短暂留白闪。
- 阵眼：太极/法阵水墨纹样缓慢旋转，掉血时纹样墨色变淡、龟裂。

## 4. UI 规范

- 字体：思源宋体/类宋体衬线为主，标题可用书法体；正文保证小屏可读。
- 控件：纸面卡片 + 墨线描边 + 朱砂印章式按钮/角标。
- 布局：竖屏单手，主操作区在下半屏（拇指可达，呼应底层逻辑一“摸鱼单手”）。
- 动效：转场用墨晕开合/卷轴展开，避免廉价滑动。
- 信息密度：留白优先，单屏不堆叠超过玩家当前境界已解锁的入口（呼应“不糊脸 10 个入口”）。

## 5. 实现约束（给 Codex）

- 所有颜色、特效参数（粒子数、晕染半径、持续时间、色键）走 `src/config/visual.json`，不在 shader/特效代码里写死（呼应底层逻辑三）。
- 优先用：自定义 shader（墨晕/飞白）、粒子系统（泼墨/飞溅）、后处理（纸纹/水墨描边/留白辉光）。
- 性能红线：微信小游戏首测目标 30 FPS、低端机 ≥25 FPS。特效不得为了好看牺牲帧率；高负载特效需可降级。
- 原型期即便用程序化占位，也必须呈现“水墨气质”（墨色、留白、笔触），不得用塑料感纯色块直接交付。

## 6. 视觉验收标准（截图/录像必查）

| 项 | 通过标准 |
|---|---|
| 整体气质 | 一眼可辨“水墨国风”，留白与墨线明显，非写实/非塑料 |
| 五行可辨 | 5 系符与特效用色符合色板，玩家能凭颜色区分五行 |
| 特效母题 | 至少火/水/木/金/土各自的水墨/泼墨/符箓表现已落地并可在录像中看到 |
| 阵眼反馈 | 掉血/告急有水墨化的可视反馈（变淡/龟裂/朱砂描边） |
| 画符正反馈 | 画符成功有笔迹追光/墨花特效，给即时爽感 |
| 性能 | 满屏特效下 ≥25 FPS（低端机），≥30 FPS（目标机） |
| 可读性 | 竖屏小屏下文字与关键数值清晰可读 |

> 视觉验收必须有 `npm run quality:gate:capture` 的截图/录像证据，代码与画面冲突时以画面为准。
