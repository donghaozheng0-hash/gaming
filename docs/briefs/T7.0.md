## 3 · 基调(Claude 已定,照此绘制;产品负责人看图终拍)

**总纲:宣纸留白 + 浓墨主笔 + 朱砂点睛 + 五行晕染。参照《雾山五行》的强笔触/高对比/大面积留白,但用程序化笔触逼近,不追求逐帧插画。**

### 3.1 板块①入场界面(entry)
- 全屏宣纸底(palette.surface.paper)。
- 背景:2-3 层远山淡墨剪影(半透明 ink,层次递淡),底部留白 ≥ 1/3。
- 标题"符塔":大号粗笔画字(用系统字重叠加描边/微旋转营造书法感,不引字体),右下角落一枚小朱砂印(方形,palette 朱砂红)。
- 入口按钮:朱印方章样式,阴刻"入塔"二字;下方细墨线注"每日三局·塔无尽头"(slogan 占位)。

### 3.2 板块②战斗 HUD(hud)
- 以 16:9 框线画一个战斗画面占位(可嵌一张淡化的路径示意)。
- 顶栏(卷轴横幅):左"第 12 层"(深度)、中"第 3/7 波"、右灵机点(2/4/6 波亮起的墨点×3)。
- **R4 情报条**(顶栏下,PREP 期展示):本关怪物五行构成——五行图标横列+数量点,Boss 波位置以小朱砂旗标记。
- 底部符栏:3-4 张符牌(竖长册页卡,内画符箓笔画+五行色角标),右侧"画符"毛笔图标入口。
- 全部为静态草图,标注"非交互"。

### 3.3 板块③八怪图鉴(monsters) —— 造型语言表(逐怪照此画)
canvas 程序化水墨剪影:**墨团为体、笔触勾边、五行色只在剪影内晕染一处**(读 palette 对应元素色)。
| monsterId | 特征形 |
|---|---|
| normal_yaobing | 基本墨团直立,一笔重顿的头部 |
| swarm_xiaoyao | 三小团簇拥,错落高低 |
| fast_yao | 低伏前倾,身后两道拖尾流线(飞白) |
| armored_yao | 方厚块身形,横向三笔甲片 |
| shield_yao | 常规墨团+体外一圈半透明环晕 |

| chapter_boss | 大墨团(约常规×2.2),体表朱砂裂纹,底部威压晕圈 |
每图下标注:id / 中文名 / 特征一句话(与导出的 `MONSTER_SKETCH_SPECS` 同源)。

### 3.4 板块④画符示意(draw)
- 一块符纸区,画一条示意轨迹(雷符折线样例)。
- 三档评分反馈样式并排:<50 墨迹涣散(灰淡)、≥50 成符(墨定形)、≥80 笔锋朱边(+20% 标注)、≥95 金光描边(完美)。
- 标注文字:"评分阈值 50/80/95,来自 balance 配置——本页仅样式示意"。

## 4 · 关键设计(为何这样定)
1. **草图先行**是产品要求:T7 正式契约(交互+视觉参数进 config)在用户看图拍板后才签发——本批的"验收终点"是用户点头,门禁只保证工程干净。
2. **确定性绘制**让"拍板的图"与"日后回看的图"逐像素一致,评审结论可追溯。
3. **色只读 palette** 保证草图与游戏同色源——拍板的是造型语言,不是一套要重调的颜色。
4. **MONSTER_SKETCH_SPECS 与 config 八怪 id 钉死**,防图鉴与数值表漂移(少画/多画/改名都会红)。
