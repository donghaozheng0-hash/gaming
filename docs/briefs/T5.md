## 3 · 交付物(精确接口)

### 3.1 `src/render/mapping.ts`
```ts
export interface WorldPoint { x: number; y: number; z: number }
export function canvasToWorld(
  p: { x: number; y: number },
  opts: { canvas: { widthUnits: number; heightUnits: number }; worldUnitsPerCanvasUnit: number },
): WorldPoint;
// 世界X = (p.x - widthUnits/2) * u；世界Y = 0(纸面)；世界Z = (heightUnits/2 - p.y) * u
```
纯函数、不 import Babylon(返回普通对象)。

### 3.2 `src/render/battleScene.ts`
```ts
import type { Engine } from "@babylonjs/core";
import type { GameConfig } from "../config";
import type { GeneratedMap } from "../game/battle/map/MapGenerator";
export function createBattleScene(engine: Engine, deps: { config: GameConfig; map: GeneratedMap }): Scene;
```
场景图契约(尺子逐条机器验证)：
- `scene.clearColor` = `visual.palette.surface.paper`(宣纸底)。
- 名为 `"paper"` 的地面节点(尺寸 = 画布映射尺寸 + `paperMarginCanvasUnits` 边距;色调可用 paper 色,禁贴图)。
- 每条路线 `route-${i}`：沿 `map.routes[i]` 的墨线(建议 CreateLines/CreateTube + ink.main 色),**节点 `metadata.points` 必须存放该路线的画布坐标数组**(与 `map.routes[i]` 深等,供尺子与后续 T6 寻路对账)。
- `"core"` 阵眼节点：位于 `canvasToWorld(路线终点)`,半径 `coreRadiusCanvasUnits`(换算世界单位),用 ink 色系(墨色纹样占位)。
- 每个开放格 `slot-${slotTypeId}`：位于 `canvasToWorld(slot.position)`,半径 `slotRadiusCanvasUnits`;`metadata = { slotTypeId, element }`;材质主色(diffuse/emissive/albedo 任一)= `visual.palette.elements[element].primary`(五行色符位,一眼可辨)。
- 相机：ArcRotate,参数全读 `visual.scene.camera`(alphaDeg/betaDeg/radiusWorldUnits,注意度→弧度换算),俯视构图让整张纸入画。
- 光照:简单 Hemispheric 即可,强度可写 visual.scene(若需新键,命名自定,schema 同步校验;尺子不锁光照)。

### 3.3 `src/config/visual.json` + `schema/visual.ts`
顶层新增(键名钉死,数值你按构图审美定,schema 校验全为正数)：
```json
"scene": {
  "worldUnitsPerCanvasUnit": 0.01,
  "camera": { "alphaDeg": ..., "betaDeg": ..., "radiusWorldUnits": ... },
  "routeWidthCanvasUnits": ...,
  "slotRadiusCanvasUnits": ...,
  "coreRadiusCanvasUnits": ...,
  "paperMarginCanvasUnits": ...
}
```
schema:`assertExactKeys` 收 `"scene"`,各字段数值校验(违反即抛,禁静默兜底)。GameConfig 的 visual 类型同步。

## 4 · 关键设计(为何这样定)
1. **场景图契约 = 渲染层的黄金算例**：截图人眼复核主观,节点命名/metadata/材质色的机器断言给 L3 兜底,也给 T6(怪沿 metadata.points 走、塔占 slot)钉好接线。
2. **颜色只经 visual.json**:视觉是品牌护城河(逻辑四),换色调不应动代码;静态扫描把这条变成机器纪律。
3. **NullEngine 可构建**=渲染逻辑与 GPU 解耦,CI/无头环境可验;这就是为什么禁 DynamicTexture。
4. **mapping 纯函数**:画布(数据真相源)与世界(表现)的唯一换算点,T6 怪物移动复用同一函数,杜绝两套换算。
