## 3 · 交付物(精确接口)

### 3.1 `src/game/battle/map/rng.ts`
```ts
export function createRng(seed: number): () => number;
```
- 推荐 mulberry32(结构魔数逐行 `// iso-ok: PRNG 结构常数`)。
- 返回 `[0,1)` 均匀数；同 seed 序列完全一致；seed=0 也必须确定可复现。

### 3.2 `src/game/battle/map/MapGenerator.ts`
```ts
import type { GameConfig } from "../../../config";

export interface Vec2 { x: number; y: number }
export interface OpenSlot { slotTypeId: string; element: string; position: Vec2 }
export interface GeneratedMap {
  poolId: string;
  templateId: string;
  archetype: string;
  routes: Vec2[][];       // 逐点等于配置模板的 routes —— 运行时只【选择】,不发明坐标
  openSlots: OpenSlot[];  // 恰 randomization.openSlotCount 个,slotTypeId 互异
}
export function generateMap(opts: { config: GameConfig; seed: number; poolId?: string }): GeneratedMap;
```
生成规则(全部随机数来自 `createRng(seed)` 单一流)：
1. `poolId` 缺省取 `config.maps.mapPools[0].id`；找不到池 → 抛错(不兜底)。
2. **选模板**：用 rng 从池的 `pathTemplates` 均匀抽 1 个。
3. **开格**：从模板 `candidateSlots` 用 rng 做部分 Fisher–Yates,取恰 `randomization.openSlotCount` 个(互异)。
4. **五行**：每个开放格用 rng 从 `randomization.elementPool` 均匀抽 1 个(可重复)。
5. `routes` 直接取模板配置(深拷贝或原样引用均可,但输出需与配置深等)。
- 纯函数：同 (config,seed) 永远同结果；无模块级可变状态。

### 3.3 配置扩展 `src/config/maps.json` + `src/config/schema/maps.ts`
- 顶层新增 `"canvas": { "widthUnits": 1000, "heightUnits": 1000 }`。
- 每个 `pathTemplates[]` 增补(并**删除** `candidateSlotTypeIds`)：
  - `"routes": Vec2[][]` — 长度 === `routeCount`；每条 ≥2 点；所有坐标 ∈ 画布：
    - `straight_pressure`(1 条)：近似直线横穿(可 2–3 点)，终点为阵眼；
    - `zigzag_path`(1 条)：**≥4 点**(至少两个拐点)的折线；
    - `dual_entry_merge`(2 条)：**两个入口点不同**(如上/下两侧)，**最后一点(阵眼)完全相同**(数值上逐字段相等)。
    - 三条模板的阵眼位置建议统一放画布右侧(如 x≈900)，方便 T5 渲染。
  - `"candidateSlots": [{ "slotTypeId": "...", "position": {"x":..,"y":..} }]` — 沿用原 `candidateSlotTypeIds` 的格位类型集合,每个补上贴近路线的画布坐标；互异；每模板 **≥ openSlotCount+2 个**(现状 8/9/10 满足)；`slotTypeId` 必须 ∈ 顶层 `candidateSlotTypes`。
- schema 同步扩展校验(风格与现有一致,违反即抛错)：canvas 数值为正；routes 长度===routeCount、每条≥2 点、坐标∈画布；candidateSlots 互异、引用可解析、position∈画布。
- **不改任何平衡数值表**(balance/infinite/dungeons/fatigue 等一律不碰)。

## 4 · 关键设计(为何这样定)
1. **几何进配置、逻辑进领域**：`src/game/battle` 受零裸数字门禁扫描,坐标写代码里必挂 isolation;且 T5 渲染/未来关卡编辑都要直接消费同一份几何数据(单一真相源)。
2. **单一 rng 流 + 注入 seed = 回放与埋点根基**：将来无限模式每层的地图 seed 会记录进运行存档,同 seed 必须逐字节复现(与 T3 定步长同一条确定性铁律)。
3. **运行时只选择、不发明坐标**：MapGenerator 的输出 routes 必须与配置深等,这让尺子能机器验证"几何未被代码篡改"。
4. **candidateSlots 带坐标替代 candidateSlotTypeIds**：格位类型(打法语义)与格位实例(模板内坐标)分离——类型表不动,模板决定这张图上哪些位置可用。
