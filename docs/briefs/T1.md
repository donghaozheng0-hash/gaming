## 交付物

在 `src/config/` 下建立(JSON 为纯数据，TS 为类型+校验+服务)：

```
src/config/
  schema/            # 每类配置的 TS 类型 + 运行时 validator(缺字段/类型错/坏引用即抛错)
  balance.json       # 战力/伤害/承伤公式常数(ATK×6+HP×0.6+DEF×3、减伤K=600、相生1.3/克制1.5/品质倍率、爬塔底数1.15 等)
  runes.json         # 10 张符模板
  monsters.json      # 8 类怪模板(以 R 为系数)
  levels.json        # 20 关
  waves.json         # 普通/Boss 波次模板
  maps.json          # 路径模板池 + 候选格位 + 随机开放规则
  fusion.json        # 融合配方 + 合成元素克制
  cultivation.json   # 境界基础三围表、五行修为升级表
  economy.json       # 资源流向、副本次数、突破材料
  progression.json   # 功能解锁表
  visual.json        # 五行色板/特效键/UI token(键值参照 docs/VISUAL_DIRECTION.md)
  ConfigService.ts   # 统一加载→校验→Object.freeze；业务只向它取数
  index.ts           # 对外导出(loadGameConfig)
```

## 硬约束 / 红线(违反即 FAIL)

- 配置层只有纯数据/类型/校验，**不得出现游戏逻辑分支或任何 `@babylonjs` 引入**。
- 公式常数也必须是**具名配置项**放进 `balance.json`，不得散落。
- `ConfigService` 启动时加载全部表并 `Object.freeze`；**字段缺失/类型错误/交叉引用不存在(如 levels 引用了不存在的 monster/map/wave/rune) → 加载即抛错，禁止静默兜底默认值**。
- 不要在 `src/game/**` 写任何业务实现或具体平衡数字(本批不碰领域逻辑)。
