## 5 · 交付物 B：新增 `infinite.json` + `schema/infinite.ts`

### 5.1 `src/config/infinite.json`(精确如下)

```json
{
  "depthWall": {
    "baseRequiredPower": 2000,
    "depthGrowth": 1.07
  },
  "monsterCoefficients": {
    "commonHp": 0.15,
    "commonAtk": 0.030,
    "bossHp": 3.20,
    "bossAtk": 0.070,
    "eliteHp": 0.75,
    "eliteAtk": 0.045
  },
  "bossRotation": {
    "bandLevels": 10,
    "recurEveryNLevels": 3,
    "recurAtWaveOf7": 5
  },
  "mapRandom": {
    "elementsPerLevel": { "min": 2, "max": 3 },
    "effectivePowerVariance": 0.15
  },
  "experience": {
    "baseAtDepth1": 100,
    "growth": 1.07
  }
}
```

> 设计说明(供你理解，不必写进数据)：故识精英(前段 Boss 复现)**重铸为当前深度精英档**，直接复用 `monsterCoefficients.eliteHp/eliteAtk`，故 `bossRotation` 不再单存精英系数——**避免双真相源**。

### 5.2 `schema/infinite.ts` 的 `validateInfiniteConfig`

- 顶层 `assertExactKeys`: `["depthWall","monsterCoefficients","bossRotation","mapRandom","experience"]`
- 各子块 exactKeys 与上方 JSON 完全一致，叶子 `assertNumber`；`elementsPerLevel` 用 `{min,max}` 形状(可复用 balance.ts 里 `validateMinMax` 的写法，自己在本文件实现一份即可)。
- **有牙校验(必须加)**：
  - `experience.growth === depthWall.depthGrowth`，否则抛错 `[config] infinite.experience.growth must equal depthWall.depthGrowth (v4: exp 与 R 同指数)`。
  - `mapRandom.elementsPerLevel.min >= 1 && min <= max`，否则抛错。
  - 所有系数 `> 0`。
